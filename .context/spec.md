# Nickmark 拡張機能 仕様書

## 1. 基本コンセプト
ニックネーム（別名）を使ってアドレスバーからブックマークへ瞬時にアクセスするための、キーボード操作特化型 Chrome 拡張機能。
マウス移動を最小限にしたいエンジニアやパワーユーザーをターゲットとする。

## 2. 主要機能

### A. ナビゲーションモード（通常時）
アドレスバー（オムニボックス）でキーワード `nm` + スペースの後にニックネームを入力して起動する。

- **入力例**: `nm dev`
- **挙動**: 
  - `dev` に紐づく URL をインクリメンタルに候補表示し、Enterキーで対象 URL へ遷移する。
  - **デフォルト候補**: 検索結果が存在する場合、デフォルト候補（一番上の行）には「Search for: XXX」等の案内が表示される。そのままEnterを押すと、背後で自動的にスコアが最も高い候補URLが選択され遷移する。
  - **候補表示**: 実際のURL候補は2行目以降に表示され、矢印キーやTabキーで選択・補完が可能。候補が1つのみになった場合は、デフォルト候補が「Go to XXX: YYY」という案内に切り替わる。
- **ソート順**: 
  同じニックネームに複数のURLが登録されている場合、以下の順でソートされる。
  1. `score`（指数減衰スコア）の降順
  2. スコアが同一の場合は `url` の昇順

### B. コマンドモード（ `:` プレフィックス ）
オムニボックス内でコロン `:` から始まる文字列を入力することで、拡張機能自体の操作を行う。

- **`:add [name] [title]`**: 現在開いているアクティブなタブを、指定したニックネームで登録する。2個目の引数 `[title]` は省略可能で、省略時はタブのタイトルが自動的に使用される。ニックネームが未指定の場合はエラー通知を表示する。
- **`:list`**: 拡張機能のオプション画面（管理画面）を開く。
- **`:rm [name] [url(任意)]`**: 指定したニックネームをストレージから削除する。
  - ニックネームに複数のURLが登録されている場合、サジェスト候補に各URLが一覧表示される。候補から特定のURLを選択してEnterを押すことで、そのURLのみを個別に削除できる。
  - 複数のURLが存在する状態でURLを指定せずに実行した場合、誤削除防止のため削除は中断され、候補から選択するよう通知が表示される。
  - ニックネームが未指定、または存在しない場合はエラー通知を表示する。
- **補完・デフォルト候補**: コマンド入力中、一番上の行には「Search commands for: XXX」や「Run command: XXX」といった案内が表示される。コマンドが入力途中の状態でそのままEnterを押した場合、候補リスト内で最も上位にあるコマンドが自動的に補完されて実行される（例: `:a` でEnterを押すと `:add` が実行される）。
### C. 通知・フィードバック
ブラウザ標準の通知（Notifications API）の代わりに、独自のトースト通知システムを使用する。

- **通常ページでの表示**: 
  `activeTab` と `scripting` API を使用し、現在開いているページの上部中央に控えめなトースト通知を注入・表示する。
- **特殊ページ（ブラウザ標準ページ）での挙動**:
  `chrome://newtab` などのスクリプト注入が禁止されているページでは、自動的にオプション画面（`options.html`）へ遷移し、画面内で通知を表示する。
- **デザイン**: ダークテーマを基調とした、視認性が高くかつ作業を邪魔しないデザインを採用。

### D. データ管理とストレージ
- `chrome.storage.local` を使用した高速なデータアクセス。
- `unlimitedStorage` 権限により、大量のブックマークデータ保存に対応。
- オプション画面からの UI を用いた追加・削除、およびエンジニア向けの JSON 一括インポート / エクスポート機能。

## 3. 指数減衰アルゴリズム
「直近でよく使っているもの」が上位にサジェストされるよう、遷移が行われるたびにスコアを自動調整する。

- スコア更新ロジック: `Score_new = (Score_old * 0.95 ^ elapsedDays) + 1.0`
- `elapsedDays`: 前回使用時から現在までの経過日数（1日 = `1000 * 60 * 60 * 24` ms として計算）
- 新規登録時の初期スコアは `1.0` とする。

## 4. 拡張機能権限 (Manifest V3)
- `omnibox`: アドレスバー操作用
- `storage`: データ保存用
- `unlimitedStorage`: 容量制限解除
- `tabs`: 現在のタブの取得・新規タブでのページ遷移用
- `activeTab`: 操作中のタブに対する一時的なアクセス権限（通知の表示用）
- `scripting`: ページ内へのトースト表示スクリプト実行用
