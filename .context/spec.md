# Nickmark 拡張機能 仕様書

## 1. 基本コンセプト
ニックネーム（別名）を使ってアドレスバーからブックマークへ瞬時にアクセスするための、キーボード操作特化型 Chrome 拡張機能。
マウス移動を最小限にしたいエンジニアやパワーユーザーをターゲットとする。

## 2. 主要機能

### A. ナビゲーションモード（通常時）
アドレスバー（オムニボックス）でキーワード `nm` + スペースの後にニックネームを入力して起動する。

- **入力例**: `nm dev`
- **挙動**: 
  - `dev` に紐づく URL をインクリメンタルに候補表示し、Enterキーで対象 URL へ遷移する。
  - **デフォルト候補**: 検索結果が存在する場合、デフォルト候補（一番上の行）には「Search for: XXX」等の案内が表示される。そのままEnterを押すと、背後で自動的にスコアが最も高い候補URLが選択され遷移する。
  - **候補表示**: 実際のURL候補は2行目以降に表示され、矢印キーやTabキーで選択・補完が可能。候補が1つのみになった場合は、デフォルト候補が「Go to XXX: YYY」という案内に切り替わる。
- **ソート順**: 
  同じニックネームに複数のURLが登録されている場合、以下の順でソートされる。
  1. `score`（指数減衰スコア）の降順
  2. スコアが同一の場合は `url` の昇順

### B. コマンドモード（ `:` プレフィックス ）
オムニボックス内でコロン `:` から始まる文字列を入力することで、拡張機能自体の操作を行う。

- **`:add [name] [title]` (エイリアス: `:a`)**: 現在開いているアクティブなタブを、指定したニックネームで登録する。2個目の引数 `[title]` は省略可能で、省略時はタブのタイトルが自動的に使用される。ニックネームが未指定の場合はエラー通知を表示する。
- **`:addall [name]` (エイリアス: `:aa`)**: 現在のウィンドウで開いているすべてのタブ（`chrome://` などを除く）を、指定したニックネームで一括登録する。ニックネームが未指定の場合はエラー通知を表示する。
- **`:ls`**: 拡張機能のブックマーク一覧画面（管理画面）を開く。
- **`:prefs`**: 拡張機能の設定画面を開く。使用言語の切り替えなどが可能。
- **`:help`**: ヘルプ・ドキュメントページを開く。言語設定に応じて適切な URL（日本語版: `https://sawara.me/extensions/nickmark`, 英語版: `https://sawara.me/en/extensions/nickmark`）が別タブで表示される。
- **`:open [name]` (エイリアス: `:o`)**: 指定したニックネームに紐づくすべての URL を、新しいバックグラウンドタブで一斉に開く。
  - **通知**: 成功時は作業の邪魔にならないようトースト通知を表示しない。ニックネームが見つからない等のエラー時のみ通知を表示する。
- **`:rm [name] [url(任意)]`**: 指定したニックネームをストレージから削除する。
  - ニックネームに複数のURLが登録されている場合、サジェスト候補に各URLが一覧表示される。候補から特定のURLを選択してEnterを押すことで、そのURLのみを個別に削除できる。
  - 複数のURLが存在する状態でURLを指定せずに実行した場合、誤削除防止のため削除は中断され、候補から選択するよう通知が表示される。
  - ニックネームが未指定、または存在しない場合はエラー通知を表示する。
- **補完・デフォルト候補**: コマンド入力中、一番上の行には「Search commands for: XXX」や「Run command: XXX」といった案内が表示される。コマンドが入力途中の状態でそのままEnterを押した場合、候補リスト内で最も上位にあるコマンドが自動的に補完されて実行される（例: `:a` でEnterを押すと `:add` が実行される）。

### C. 設定と多言語対応
拡張機能全体の表示言語をカスタマイズできる。

- **言語切り替え**: 設定画面（`preferences.html`）から「日本語」または「英語」を選択可能。
- **優先順位**: ユーザーが明示的に設定した言語が優先され、未設定の場合はブラウザの標準言語設定（`chrome.i18n.getUILanguage()`）が使用される。
- **反映範囲**: 管理画面、設定画面、およびバックグラウンドから実行されるトースト通知（登録完了メッセージ等）のすべてに即座に反映される。

### D. 通知・フィードバック
ブラウザ標準の通知（Notifications API）の代わりに、独自のトースト通知システムを使用する。

- **通常ページでの表示**: 
  `activeTab` と `scripting` API を使用し、現在開いているページの上部中央に控えめなトースト通知を注入・表示する。
- **特殊ページ（ブラウザ標準ページ）での挙動**:
  `chrome://newtab` などのスクリプト注入が禁止されているページでは、自動的にブックマーク一覧画面（`list.html`）へ遷移し、画面内で通知を表示する。
- **デザイン**: ダークテーマを基調とした、視認性が高くかつ作業を邪魔しないデザインを採用。

### E. データ管理とストレージ
- **ハイブリッドストレージ**: 高速なアクセスが可能な `chrome.storage.local` と、デバイス間同期が可能な `chrome.storage.sync` を切り替えて使用可能。
- **容量**: `unlimitedStorage` 権限により、ローカル保存時は実質無制限のデータ保存に対応。同期時は Google の制限（合計 100KB）に従う。
- **バックアップ**: エンジニア向けの JSON 一括インポート / エクスポート機能を搭載。

### F. Google アカウント同期
Google アカウントに紐づけて、複数の PC 間でブックマークデータを自動同期する機能。

- **同期の切り替え**: 設定画面（`preferences.html`）から「Google アカウント同期」を有効にできる。
- **データのチャンク化**: `chrome.storage.sync` の 1 項目あたり 8KB という制限を回避するため、データを 8000 バイトずつのチャンクに分割して保存し、読み込み時に自動で結合・復元する。
- **データ移行と競合解決**:
  - 同期の ON/OFF 切り替え時に、データの移行先を選択する「データ移行ダイアログ」を表示する。
  - ローカルとクラウドの両方にデータが存在する場合、それぞれの「最終更新日時（ブックマークの作成・使用日時）」と「アイテム数」を比較表示し、ユーザーが優先するデータを選択できる。
  - トグルの操作は物理的にインターセプト（遮断）されており、ダイアログでユーザーが「承認」するまではステートが変更されない。
- **容量制限チェック**: 同期を有効にする前にローカルデータのサイズをチェックし、100KB を超えている場合は有効化を制限してデータの削減を促す。
- **設定の独立性**: 「同期の有効/無効設定」および「言語設定」は常に `chrome.storage.local` に保存され、各デバイスで独立して管理される。

## 3. 指数減衰アルゴリズム
「直近でよく使っているもの」が上位にサジェストされるよう、遷移が行われるたびにスコアを自動調整する。

- スコア更新ロジック: `Score_new = (Score_old * 0.95 ^ elapsedDays) + 1.0`
- `elapsedDays`: 前回使用時から現在までの経過日数（1日 = `1000 * 60 * 60 * 24` ms として計算）
- 新規登録時の初期スコアは `1.0` とする。
