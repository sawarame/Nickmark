# デザインルール (Design Rules)

主にオプション画面（管理画面）のデザインと UI 実装に関するルールを定めます。

## 1. UI フレームワーク
オプション画面の UI には **Vue 3** と **PrimeVue (v4)** を採用する。
プレーンな HTML / CSS による実装は避け、可能な限り PrimeVue のコンポーネントを利用して構築する。

## 2. テーマとスタイリング
- **テーマ**: `@primeuix/themes` を利用し、`Aura` テーマをデフォルトとする。
- **アイコン**: `primeicons` を使用する（例: `pi pi-plus`, `pi pi-trash`）。
- **スタイリング**: 
  - コンポーネントの配置やレイアウトの調整には、CSS の Flexbox や Grid を利用する。
  - Vue の `<style scoped>` を用いてコンポーネント内にスタイルをカプセル化する。
  - 背景色やテキストカラーは PrimeVue の CSS 変数（例: `var(--p-surface-100)`, `var(--p-text-color)`）を利用し、テーマに追従させる。

## 3. 使用する主なコンポーネント
- **レイアウト**: `Card` を用いて、セクションごと（登録、一覧、インポート/エクスポート）に情報をグループ化する。
- **データ表示**: `DataTable` と `Column` を使用して、ブックマーク一覧を見やすく表示し、ソート機能 (`sortable`) やページネーション (`paginator`) を活用する。
- **入力フォーム**: `InputText` および `Textarea` を使用する。
- **アクション**: `Button` を使用し、破壊的なアクション（削除）には `severity="danger"`、副次的なアクションには `severity="secondary"` を指定して視覚的に区別する。
- **ユーザーフィードバック**: `Toast` コンポーネントを利用し、登録完了、削除完了、エラー発生時などに画面右上に通知を表示する。

## 4. ユーザビリティ
- フォームには適切な `placeholder` や `label` を設定する。
- 入力エラー（例：ニックネームやURLが空の場合、重複するURLの登録）には Toast で明確なエラーメッセージを提示し、不正な操作をブロックする。
