# 開発ワークフロー (Development Workflow)

## 1. 開発の準備
1. プロレクトリのルートで `npm install` を実行し、依存パッケージをインストールする。
2. アイコンが生成されていない場合は `node generate_icons.js` を実行し、`Nickmark/icons/` 以下にアイコンを用意する。

## 2. ソースコードの編集
- ロジックや UI の変更は常に `src/` ディレクトリ内のファイル（`.ts`, `.vue`）に対して行う。
- `Nickmark/js/` 以下のファイルはビルド成果物であるため、直接編集しない。
- 拡張機能の設定変更（権限追加など）は `Nickmark/manifest.json` を編集する。

## 3. ビルドの実行
ソースコードの変更が完了したら、以下のコマンドを実行してビルドを行う。

```bash
npm run build
```

これにより、以下の処理が自動で行われます。
- `Nickmark/js/` フォルダのクリーンアップ。
- `Nickmark.zip` の削除。
- Webpack による TypeScript および Vue ファイルのコンパイルとバンドル（出力先: `Nickmark/js/`）。
- `Nickmark/` フォルダ全体の zip 圧縮 (`Nickmark.zip` の生成)。

## 4. ブラウザでの動作検証
1. Google Chrome を開き、`chrome://extensions/` にアクセスする。
2. 右上の「デベロッパー モード」をオンにする。
3. 「パッケージ化されていない拡張機能を読み込む (Load unpacked)」ボタンをクリックする。
4. このプロジェクト内の `Nickmark` フォルダを選択して読み込む。
5. 変更を加えた場合は、拡張機能のカードの右下にある「再読み込み (更新)」アイコンをクリックして変更を反映させる。
6. オムニボックスで `nm ` と入力し、機能やコマンド（`:add`, `:ls` など）が正常に動作するか確認する。

## 5. デバッグ
- バックグラウンドスクリプトのログやエラーは、拡張機能管理画面の「Service Worker」というリンクをクリックして DevTools を開き、Console タブで確認する。
- オプション画面のログや UI の構造は、オプション画面を開いたタブで右クリック ＞ 「検証 (Inspect)」から DevTools を開いて確認する。
